# 基本方針

要望、要件を伝えられた場合、以下の手順で作業を行う。
確認、精査、実装方針検討、実装方針再確認、設計ドキュメント作成または修正、コーディング方針検討、コーディング方針再確認、コード作成、コード自己レビュー（指摘がなくなるまで繰り返し）

必ず最初にドキュメントの作成、修正を行ってから、それを実装行う。
実装内容に基づいて、辻褄を合わすような設計ドキュメント修正は禁止とする。あくまで、要望、要件を満たすためのドキュメントを作成する。

# 設計について
要望、要件を実現するための設計ドキュメントを作成する。
ｰ 基本設計書：言語を中心として、図表程度を利用しながら、設計の概要、大まかな方針を表現する資料。
- 詳細設計書：基本設計書の内容を具体的にコードでどのように実現するか、詳細に説明した資料。

# レビュー結果を受け取った際の修正について

レビュー結果を受け取った際は以下の手順で作業を行う。
確認、精査、指摘内容の妥当性確認、修正要否判断、修正方針検討、修正方針再確認、修正、修正内容の自己レビュー（指摘がなくなるまで繰り返す）
